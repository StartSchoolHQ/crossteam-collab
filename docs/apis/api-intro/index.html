API

Introduction to APIs​

Once again with the restaurant analogy. You (the customer) want food from the kitchen, but it's not acceptable for you to just walk into the kitchen and cook it yourself. Instead, you tell the waiter what you want, the waiter goes to the kitchen, and brings back your food.

An API (Application Programming Interface) is like that waiter - it's the messenger between two systems that allows them to communicate without you needing to know how the kitchen (backend) works.

What is an API?​

An API is a set of rules that allows different software applications to talk to each other. It defines:

  • What requests you can make
  • How to make those requests
  • What format the data should be in
  • What responses you'll get back

Think of it as a contract: "If you ask me for X in this specific way, I'll give you Y back."

Why APIs matter​

APIs are everywhere in modern technology:

  • When you check the weather on your phone, the app uses a weather API
  • When you log in with Google or Facebook, you're using their authentication API
  • When you pay with a credit card online, payment APIs process the transaction
  • When you see a map on a website, it's using a mapping API (like Google Maps)

Example | Web app login​

The flow:

  1. You click "Login with Google" on a website
  2. The website's code asks Google's API: "Can this person log in?"
  3. Google shows you a login page
  4. You enter credentials
  5. Google's API tells the website: "Yes, we know this person, we have them in our data base, this is John Doe, email: john@gmail.com"
  6. You're logged in!

The website never sees your Google password - the API handles everything securely.

JSON - the language of APIs​

Most modern APIs use JSON (JavaScript Object Notation) to send data.

What is JSON?​

JSON is a text format for storing and transporting data. It's easy for both humans and computers to read.

Example:

{
  "name": "Anna",
  "age": 28,
  "city": "Riga",
  "hobbies": ["reading", "coding", "traveling"],
  "employed": true
}

JSON structure​

Objects (key-value pairs):

{ "name": "Anna", "age": 42 }

Arrays (lists):

["apple", "banana", "orange"]

Values can be:

  • Strings: "hello"
  • Numbers: 42, 3.14
  • Booleans: true, false
  • Arrays: [1, 2, 3]
  • Objects: {"key": "value"}
  • Null: null

Real API response example​

{
  "success": true,
  "user": {
    "id": 123,
    "name": "Anna Bērziņa",
    "email": "anna@example.com",
    "role": "developer"
  },
  "permissions": ["read", "write"],
  "lastLogin": "2024-10-25T10:30:00Z"
}

API authentication - proving who you are​

Most APIs require authentication to:

  • Know who's making requests
  • Protect sensitive data
  • Prevent abuse
  • Track usage

Common authentication methods​

1. API keys

Simple string that identifies you:

https://api.example.com/data?api_key=abc123xyz

Like: A membership card at a store

2. Bearer tokens (OAuth)

More secure, temporary access:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Like: A day pass to an event

3. Basic authentication

Username and password:

Authorization: Basic dXNlcjpwYXNz

Like: Showing ID at the door

API documentation - the instruction manual​

Good APIs come with documentation that explains:

What you need to know​

1. Base URL:

https://api.example.com/v1

2. Available endpoints:

GET  /users
POST /users
GET  /users/:id

3. Required parameters:

name (required, string)
email (required, string, must be valid email)
age (optional, number)

4. Example requests:

fetch('https://api.example.com/v1/users', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ name: 'Anna', email: 'anna@example.com' })
})

5. Example responses:

{
  "id": 123,
  "name": "Anna",
  "email": "anna@example.com",
  "createdAt": "2024-10-25T10:30:00Z"
}

6. Error codes and messages:

400 - Invalid email format
401 - Missing authentication token
500 - Database connection failed

Making API calls - practical example​

Let's see how to make an actual API call using JavaScript:

Example: Get user data​

// Make GET request to API
fetch('https://api.example.com/users/123')
  .then(response => {
    // Check if request was successful
    if (!response.ok) {
      throw new Error('User not found');
    }
    // Convert response to JSON
    return response.json();
  })
  .then(data => {
    // Use the data
    console.log('User name:', data.name);
    console.log('User email:', data.email);
  })
  .catch(error => {
    // Handle errors
    console.error('Error:', error);
  });

Example: Create new user​

// Prepare data to send
const userData = {
  name: 'Anna',
  email: 'anna@example.com',
  age: 28
};

// Make POST request
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your-token-here'
  },
  body: JSON.stringify(userData)
})
  .then(response => response.json())
  .then(data => {
    console.log('User created with ID:', data.id);
  })
  .catch(error => {
    console.error('Error creating user:', error);
  });

API rate limiting​

Many APIs limit how many requests you can make:

Why?

  • Prevent abuse
  • Ensure fair usage
  • Protect server resources
  • Control costs

Common limits:

  • 100 requests per hour (free tier)
  • 1000 requests per hour (paid tier)
  • 10,000 requests per day

What happens when you exceed the limit?

  • You get a 429 Too Many Requests error
  • You must wait before making more requests

Common API patterns​

Pagination​

When there's too much data to send at once:

Request:

GET /api/users?page=2&limit=20

Response:

{
  "data": [...],
  "page": 2,
  "totalPages": 10,
  "totalItems": 200
}

Filtering​

Get only what you need:

GET /api/products?category=electronics&price_max=500

Sorting​

Order results:

GET /api/users?sort=name&order=asc

Search​

Find specific items:

GET /api/products?search=laptop

Testing APIs​

Tools for testing​

1. Browser DevTools

  • Open console (F12)
  • Use fetch() to test requests
  • See network tab for all requests

2. Postman

  • Desktop app for API testing
  • Save and organize requests
  • Share with team
  • Generate code examples

3. cURL (command line)

curl https://api.example.com/users

4. Online tools

  • httpbin.org - test HTTP requests
  • jsonplaceholder.typicode.com - fake API for testing

Why APIs matter for non-developers​

Understanding APIs helps you:

For project managers​

  • Estimate timelines - "Integrating this API will take X days"
  • Understand dependencies - "We need to wait for their API to be ready"
  • Make technical decisions - "Should we build this or use an API?"
  • Communicate with developers - Speak the same language

For product owners​

  • Evaluate features - "Can we add Google login using their API?"
  • Understand limitations - "The API only allows 100 requests/hour"
  • Plan integrations - "Which third-party services should we connect?"
  • Assess feasibility - "Is there an API for that?"

For business analysts​

  • Write requirements - "System must integrate with payment API"
  • Document workflows - "User clicks button → API call → Display result"
  • Understand data flow - "Where does this data come from?"
  • Identify bottlenecks - "The API is slow, causing delays"

Next steps​

Ready to practice? Try these:

  1. Explore a public API - Pick one from the resources above
  2. Use browser console - Make a fetch() call to test it
  3. Read API documentation - Pick your favorite service and read their docs
  4. Watch API calls - Open DevTools on any website and watch Network tab
  5. Ask developers - Next time you hear "API", ask them to explain the integration

Remember: You don't need to build APIs, but understanding them makes you a better team member and more valuable professional!

APIs are the glue that holds modern software together. Now you understand how that glue works!
