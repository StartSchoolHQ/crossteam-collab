Databases

Part 2: Understanding the Complete Flow​

Now that you have all the code, let's trace what happens when a customer places an order.

The Complete Journey​

Initial state of mock database (in server.js):

// Backend database BEFORE any requests
products = [
    { id: 1, name: 'Laptop', price: 999, stock: 5 }
]
orders = []

Step-by-Step Flow​

FRONTEND (index.html + app.js)

  1. User sees the page:

    • index.html loads
    • Displays form for placing orders
    • JavaScript in app.js runs automatically

  2. Page loads products:

    • app.js calls loadProducts() function
    • Makes GET request to backend

  3. User fills in the form:

    • Product ID: 1
    • Quantity: 2
    • Customer Name: "John Doe"

  4. User clicks "Place Order" button

  5. JavaScript collects the data (app.js):

    const orderData = {
        productId: 1,
        quantity: 2,
        customerName: "John Doe"
    };

  6. JavaScript makes API call (app.js):

    fetch('http://localhost:3000/api/orders', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(orderData)
    })

⬇️ API REQUEST SENT OVER NETWORK ⬇️

POST /api/orders
Content-Type: application/json

{
    "productId": 1,
    "quantity": 2,
    "customerName": "John Doe"
}

BACKEND (server.js)

  1. Express receives the request:

    • Routes to app.post('/api/orders') handler

  2. Backend extracts data:

    const { productId, quantity, customerName } = req.body;

  3. Backend validates the data:

    • ✓ productId is present (1)
    • ✓ quantity is present (2)
    • ✓ customerName is present ("John Doe")

  4. Backend queries mock database:

    • Calls getProductById(1)
    • Finds: { id: 1, name: 'Laptop', price: 999, stock: 5 }

  5. Backend checks business rules:

    • ✓ Product exists
    • ✓ Stock available (5 >= 2)

  6. Backend calculates total:

    const total = 999 × 2 = 1998

  7. Backend writes to mock database:

    // Creates new order
    orders.push({
        id: 1,
        productId: 1,
        productName: 'Laptop',
        quantity: 2,
        customerName: 'John Doe',
        total: 1998,
        createdAt: '2024-01-15T10:30:00Z'
    });

  8. Backend updates mock database:

    // Updates product stock
    products[0].stock = 3;  // was 5, now 3

  9. Backend sends response:

    res.status(201).json({
        success: true,
        message: 'Order created successfully',
        order: { id: 1, total: 1998, ... }
    });

⬆️ API RESPONSE SENT OVER NETWORK ⬆️

HTTP 201 Created
Content-Type: application/json

{
    "success": true,
    "message": "Order created successfully",
    "order": {
        "id": 1,
        "productId": 1,
        "productName": "Laptop",
        "quantity": 2,
        "customerName": "John Doe",
        "total": 1998,
        "createdAt": "2024-01-15T10:30:00Z"
    }
}

FRONTEND (app.js)

  1. JavaScript receives the response

  2. JavaScript checks if successful:

    if (response.ok) {
        showMessage('Order placed successfully!', 'success');
    }

  3. Frontend updates the display:

    • Shows green success message
    • Reloads products (calls loadProducts() again)
    • Shows updated stock (now 3 instead of 5)
    • Clears the form

  4. User sees confirmation:

    • Green message: "Order placed successfully! Order ID: 1, Total: $1998"
    • Stock updated in product display

Final state of mock database (in server.js):

// Backend database AFTER the request
products = [
    { id: 1, name: 'Laptop', price: 999, stock: 3 }  // ← stock reduced from 5 to 3!
]
orders = [
    {
        id: 1,
        productId: 1,
        productName: 'Laptop',
        quantity: 2,
        customerName: 'John Doe',
        total: 1998,
        createdAt: '2024-01-15T10:30:00Z'
    }  // ← new order created!
]

What You Should Notice​

  1. Frontend and backend are separate: They communicate via HTTP requests, not directly
  2. Frontend sends data: Collected from user input in the form
  3. Backend validates everything: Never trusts the frontend - checks all data
  4. Backend manages data: Database operations only happen in backend (server.js)
  5. Backend sends result: Frontend doesn't know about database, only gets response
  6. Frontend updates UI: Based on the response received, shows success or error
  7. Console logs help: Both sides log what they're doing (check browser console and terminal)
