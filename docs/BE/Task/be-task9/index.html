Tips for Success​

Before You Start:

  1. Copy all four files into your IDE
  2. Read through each file carefully
  3. Add your own comments to help understand
  4. Try changing things to see what happens

While Working:

  1. Keep all files open for easy switching
  2. Follow the console.log statements
  3. Be specific in your answers with exact values
  4. Draw diagrams if helpful
  5. Think about what each side knows

When Tracing Flows:

  1. Start with the user - what do they see?
  2. Follow to frontend - what does app.js do?
  3. Cross to backend - what does server.js receive?
  4. Watch the data change - what happens in database?
  5. Follow back to frontend - what does app.js receive?
  6. End with the user - what do they see?

Understanding the Code:

  • index.html: Structure (form, buttons, containers)
  • style.css: Appearance (colors, spacing, layouts)
  • app.js: Frontend logic (collect data, send requests, update display)
  • server.js: Backend logic (validate, process, manage database, send responses)

Expected Time​

Reading and tracing scenarios: 2-3 hours If running the code: 3-4 hours (includes setup time)

Frequently Asked Questions​

Q: I opened index.html in my browser and got errors. What is wrong?

A: The backend server is not running. Either just read and trace the code on paper, or follow the setup steps to install Node.js and run both servers.

Q: Do I need to install Node.js?

A: Only if you want to actually run the code. You can complete this task by reading and tracing without running anything.

Q: Which approach should I choose?

A: Choose reading-only if you want to focus on concepts. Choose running if you are comfortable with technical setup. Both teach the same concepts.

Q: Common errors when running:

  1. node is not recognized: Install Node.js
  2. Cannot find module express: Run npm install express cors
  3. Port 3000 already in use: Change PORT in server.js to 3001
  4. Frontend shows errors: Make sure both servers are running

Q: How do I stop the servers?

A: Press Ctrl+C (Windows/Linux) or Cmd+C (Mac) in each terminal window.
